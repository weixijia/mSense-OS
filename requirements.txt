# Notes:
# - mmWave FFT (Range/Doppler/Azimuth) runs on PyTorch: CUDA on NVIDIA, CPU elsewhere
#   (incl. Apple Silicon — torch.fft is NOT implemented on MPS), NumPy fallback if torch
#   is absent.
# - ViTPose runs on CUDA > MPS (Apple) > CPU automatically.
# - The smallest model (vitpose-s) ships in ./models or auto-downloads on first run.
# - ffmpeg-python needs the ffmpeg binary: macOS `brew install ffmpeg`, Ubuntu `apt install ffmpeg`.
# - The off-GIL C receiver (core/mmwave_capture_c.py) needs the `fpga_udp` extension built
#   locally for zero-loss recording; without it, capture auto-falls back to the Python receiver.
